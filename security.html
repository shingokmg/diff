Security

diff は、入力テキストを比較処理のためにサーバーへ送信しないことを主眼にした静的ツールです。ここでは、何を守るための設計か、どこまでを守れないかを明確にします。

このページで伝えたいこと

  • diff は、アプリ自身が入力テキストを外部へ送らないことを重視して設計しています。
  • ただし、端末やブラウザ環境そのものの安全性まで保証するものではありません。

目標

  • 入力テキストと差分結果をブラウザ内だけで処理する
  • 本文処理のための外部 API を使わない
  • 入力テキストを localStorage、sessionStorage、Cookie に保存しない
  • 公開コード で実装を検証できる状態を保つ

守る範囲

  • 入力テキストはブラウザ内で比較する
  • 差分処理のためにサーバーへ本文を送らない
  • 本文処理のための外部解析 API を使わない
  • アプリ自身は入力内容を自動保存しない

実装上の根拠

  • 本文処理用の fetch / XMLHttpRequest / WebSocket を持たない
  • 本文を localStorage、sessionStorage、Cookie に保存しない
  • 不要な外部ライブラリを増やさず、静的配信で完結する構成を優先している
  • 公開コード を確認できるようにしている

守れない範囲

  • 悪意あるブラウザ拡張
  • マルウェア、キーロガー、画面録画ソフト
  • クリップボード履歴や OS の補助機能
  • Web サイト閲覧時に通常発生するアクセスログやネットワークメタデータ
  • 外部基盤の設定不備

機密性の高い文章について

diff は、入力テキストを比較処理のためにサーバーへ送らない設計です。ただし、業務上の機密文書や個人情報を扱う場合は、組織のルール、端末管理、ブラウザ拡張機能の有無なども合わせて確認してください。
